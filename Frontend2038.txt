HISTORY OF HTML:
------------------------------------------

1). BIRTH OF HTML [1991]: -
-----------------------------

>> HTML WAS CREATE BY "TIM BERNERS-LEE" IN 1991.
>> HE DEVELOPED HTML WHILE WORKING AT CERN:European Organization for Nuclear Research
>> THE FIRST VERSION CONTAINED ONLY ABOUT 18 PAGES.

--------------------------------------------------------------

2). HTML 2.0 - [1995]
>> FIRST STANDARD VERSION OF HTML.
>> INTRODUCED FORM, ALLOWING USERS TO ENTER DATA ON WEB PAGES.
------------------------
3). HTML 3.2 - [1997]
------------
>> PUBLISED BY THE WORLD WIDE WEB CONSORTIUM [W3C]
>> ADDED SUPPORTS OF TABLES, APPLETS, AND TEXT FORMMATING.
--------------------------------------------------------------
4). HTML 4.01- [1999]
-------------------------
>> IMPROVED SUPPORT OF CSS [CASCADING STYLE SHEETS].
>> ENCOURAGED SPEPERATION OF CONTENT AND PRESENTATION.
>> BECOME THE MOST WIDELY USED VERSION FOR MANY YEARS.
------------------------------------------------------------------------
5). XHTML[2000]
-----------
>> A STRICTER VERSION OF HTML BASED ON XML RULE.
>> RQUIRED PROPER CLOSING OF TAGS AND WELL-FORMED DOCUMENTS .
------------

6). HTML 5.0 [2014]
--------------------
>> MAJOR UPDATE TO HTML.
>> ADDED MULTIMEDIA SUPPORT WITHOUT PLUGINS:
	>> <audio>
	>> <video>
	>> <canvas>
>> INTRODUCED SEMANTIC ELEMENTS:
	>> <header>
	>> <section>
	>> <article>
	>> <footer>
>> Improved support for mobile and web applications.

7). MODERN HTML
----------------------
>> HTML 5 IS THE CURRENT STANDARD USED BY MODERN WEBSITES.
>> CONTINUES TO EVOLVES THROUGH UPDATES MAINTANED BY ORGANIZATIONS SUCH AS WHATWG WHATWG (Web Hypertext Application Technology Working Group) AND W3C.

4. Universal Selector:
-----------------------------------------------------------------------------------------------------------------------------------------------------------

>> Whatever attributes you write inside Universal selector it automatically apply to all your html document.



>> It is represented by Asterisk symbol [*]
---------------------------------------------------------------------------------------------------------------------------------


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Universal Selector</title>
    <style>
        /* Universal Sselector  */
        *{
            background-color: blueviolet;
            color: antiquewhite;
            font-size: 30px;
            font-family: 'Trebuchet MS', 'Lucida Sans Unicode', 'Lucida Grande', 'Lucida Sans', Arial, sans-serif;
        }
    </style>
</head>
<body>
    <h1>Universal Selector Example</h1>
    <div>Hello CSS</div>
    <p class="kl">This is a paragraph.</p>
    <p class="po">This is another paragraph.</p>
</body>
</html>

MARGIN: 
-------------
>> IT CONSISTS OF FOUR PROPERTIES:-
----------------------------------------------
a). TOP
b). RIGHT
c). BOTTOM
d). LEFT
------------------------------------------------------------------------------------------------

>> 1. IF THE MARGIN HAS FOUR VALUE.

	margin: 20px 30px 45px 56px
	>> 20px --> TOP
	>> 30px --> RIGHT
	>> 45px	--> BOTTOM
	>> 56px	--> LEFT
	>> IT MOVE CLOCK WISE DIRECTION.
--------------------------------------------------------------------------------------

>> 2. IF THE MARGIN HAS THREE VALUE.
	MARGIN = 20px 30px 40px
	>> 20px ---> TOP
	>> 30px --> RIGHT and LEFT
	>> 40px --> BOTTOM
------------------------------------------------------------------------------------------
>> 3. IF THE MARGIN HAS TWO VALUES
-------------------------
	MARGIN: 40px 50px
	>> 40px --> TOP and BOTTOM
	>> 50px --> RIGHT and LEFT
----------------------------------------------------------------------------------------
>> 4. IF THE MARGIN HAS ONE VALUE
	MARGIN: 20px
	>> 20px-- TOP
	>> 20px-- RIGTH
	>> 20px-- BOTTOM
	>> 20px-- LEFT
-------------------------------------------------------------------------------------------------

Note:

>> JavaScript has several data types, and each variable has defined scope, determining where in your code you can access that variable.

Data Types
-------------------------
Primitive Data types.
------------------------------------------------

[Primitive Data types:]
	>> 1. Number: Represents both integer and floating-point numbers.
	>> 2. String: Represents a sequence of characters enclosed in single, double or backticks.
	>> 3. Boolean: Represents a logical entity, having only two values: true or false.
	>> 4. Null: Represents the intentional absence of any object value.
	>> 5. Undefined: Represents a variable that has been declared but not assigned value.
	>> 6. Symbol [ES6]: Represents a unique and immutable value, often used as object key.
	>> 7.BigInt (ES2020): Represents integers with arbitrary precision, exceeding the limit of the number type.


* [Complex Data type/Non-Primitive]
	>> Object: Represents a collection of key-value pairs, where keys are strings and values can be of any data type.

-------------------------------------------------------------------------------------------------

>> Scope:
	>> Scope determines the accessibility of variables, functions, and objects in some particular part of your code during runtime. 

> JavaScript has the following types of scope:
----------------------------------------------
>> 1. Global Scope
	>> Variable declared outside of any function block have global scope. They can be accessed and modified from anywhere in your code.

>> 2. Function Scope [Local Scope]
	>> Variables declared within a function have function scope. They can only be accessed and modified within the function.


>> 3. Block Scope [ES6]
--------------------------------
	>> Introduced in ES6, variables declared with let and const keywords have block scope. This means it is only accessible within the block where it defined in, such as within an if statement, loop or a block of code defined by curly braces.

--------------------------------------------------------------------------------------

| Variable      | Declaration | Scope          | Accessible From            |
| ------------- | ----------- | -------------- | -------------------------- |
| `globalVar`   | `var`       | Global Scope   | Anywhere in the program    |
| `functionVar` | `var`       | Function Scope | Only inside `myFunction()` |
| `blockVar`    | `let`       | Block Scope    | Only inside the `if` block |

### Difference Between `var`, `let`, and `const`

| **Feature** | **`var`** | **`let`** | **`const`** |
|-------------|-----------|-----------|-------------|
| **Scope** | Function / Global | Block `{ }` | Block `{ }` |
| **Reassignment** | Yes | Yes | No |
| **Redeclaration** | Yes | No | No |
| **Hoisting** | Hoisted (`undefined`) | Hoisted (Uninitialized / TDZ) | Hoisted (Uninitialized / TDZ) |
| **Initial Value Required** | No | No | Yes |

// 1 var --- Function Block 

var x = 5;   // global variable 
console.log(x);
{
    var x=25;  // local variable iniside the block
    console.log(x);
}
console.log(x);  // global variable is changed because var is function scoped, not block scoped //25


// 2. let --- Function Block

let y = 10;  // global variable is 10
console.log(y);
{
    let y = 20;  // local variable inside the block is 20
    console.log(y);
}
console.log(y);  // global variable is not changed because let is block scoped //10


// 3. const --- Function Block

const z = 15;  // global constant is 15
console.log(z);
// const z = 30;  // This would cause a TypeError because z is already declared as a constant
//z = 30; // This would cause a TypeError because z is a constant and cannot be reassigned
{
    const z = 25;  // local constant inside the block is 25
    console.log(z);
}
console.log(z);  // global constant is not changed because const is block scoped //15
